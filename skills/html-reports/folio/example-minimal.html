Preface

Introduction & reading guide#

Folio is the book-shaped archetype in the html-reports library. It is designed for documents where the reader will hop between many independent sections — per-person scorecards, framework references, audit findings — and wants a persistent table of contents to navigate. The TOC on the left stays fixed, the content scrolls; as you scroll, the active section highlights in the sidebar.

This example renders against the same template a real report would use, so every feature you see here is something you can rely on in production. The sidebar uses grouped navigation (three groups, click a group label to collapse it); the alternative flat list is one fewer wrapper.

Preface

Visual identity#

The palette is intentional. Parchment cream in the sidebar, warm off-white in the body, deep indigo for accents and active TOC links, and a gold/ochre rule as the signature device — a 40-80px horizontal mark under titles. Titles are set in a system serif stack (Iowan Old Style → Palatino → Georgia), which gives Folio a manuscript feel without ever requesting a webfont.

Why serifs for headings, sans for body?

Serifs on titles cue "long-form document" the way they do for books and journals. Sans body keeps long-scroll screen reading comfortable. The contrast also makes the section hierarchy visible at a glance.

Reserved tones

  • Deep indigo (#1d2960) — accents, links, active states.
  • Brick / iron-oxide (#a8442a) — warnings, regression callouts.
  • Forest green (#1f6f4a) — wins, strengths.
  • Gold (#c9b88a) — rules and decorative dividers only.

Code blocks#

Inline code gets a subtle parchment chip. Block code uses the gold rule as a left border — a visual rhyme with section titles. Long lines scroll horizontally rather than wrap.

A deliberately long section title that should wrap gracefully without breaking the layout or pushing past the 28ch measure#

The title above is capped at 28ch and wraps via word-wrap: break-word; the gold underline rule stays anchored to the left at 40px. The TOC link for this section uses an equally long label — observe that the sidebar entry wraps to two lines without breaking alignment, and the active border stays a clean 3px on the leftmost edge.

Long-read section#

This section exists to demonstrate body comfort at length. Body type is set at 16px with 1.6 line-height, in a system sans (Inter on supported platforms, San Francisco on macOS, Segoe on Windows). The measure is capped at 68ch — wider than the 64ch of a typical book, narrower than full-width — to keep the eye-saccade distance reasonable on a 27-inch monitor without feeling cramped on a 13-inch laptop.

Drop caps mark the first paragraph of every section. They are set in the indigo accent color, in the same serif as the titles, at 3.1× the body size, floated left over two-and-a-half lines. On mobile they drop to 2.6× so the first paragraph doesn't get crowded out.

Tables, callouts, code blocks, and pre-formatted text all respect the same 68ch ceiling and break gracefully on print: section breaks fall before each top-level <article>, headings stay glued to their following paragraph, callouts and code blocks never split across pages, and orphans/widows are set to 3 lines each.

Active-section highlighting

The TOC tracks the topmost visible section using a single IntersectionObserver with a rootMargin of -15% 0px -65% 0px. As you scroll, the active link transitions over ~150ms (matching the design-system motion token) and the URL hash updates via history.replaceState — so copying the URL gives you a deep link to the section you were just reading.

Keyboard & screen reader

Press Tab from the top of the page and the skip-to-content link appears in the top-left corner. From there, every TOC entry is reachable with Tab; the active focus ring uses the indigo accent at 2px with a 3px offset. Group labels are real <button> elements with aria-expanded; their aria-controls points at the group's items. Section anchors are <article> elements labelled by their <h2>, so a screen reader announces "Article, Heading level 2, Long-read section" when you land on one.

Print preview

Open this file and print to PDF — the sidebar disappears, the gold rules turn gray, link URLs are expanded inline in 9pt, and each section begins on its own page. Margins are set in millimeters via @page so the output is consistent across browsers.
