## Design decisions worth JD's review

- **The cautionary tale is the spine, generalized, not named.** The "4 crashed sites" story
  is told as *a member's* experience without naming Justin Day, and generalized to the rule
  "don't give your automation a write path into a client's live property." Keeps it teaching,
  not gossip, and applies to every member — not just the WordPress case.
- **One-way data flow is the core safety claim.** I framed "read-only" as a *property of the
  architecture* (a static file the client views; no login into your stack; no write path into
  their site), not a permission you set. That's the honest version of "read-only by
  construction" and it's what separates this from a plugin/portal.
- **Single injection point by design.** The template changes exactly one fenced JSON block per
  run (`DASHBOARD_DATA:START/END`); CSS + render JS never change. That's deliberate so an agent
  can regenerate it reliably every run without touching markup — the determinism-pattern hook.
  The JSON lives inline as a `<script type="application/json">` island (not a `fetch`) so the
  file stays truly single-file: opens by double-click, survives being emailed as an attachment,
  and can't call home.
- **"On the page vs. internal" table is the load-bearing member lesson.** The two lines I'd
  defend hardest: never expose the tool/vendor stack (hands the client a shopping list to cut
  you out), and never leak cost/margin (reframes your value as your bill). Per the hard rules I
  wrote it generically — "your SEO platform's MCP" — with no SA product/endpoint named.
- **Rung call: L5–L6.** L5 because the page is regenerated by an always-on run (it lives in the
  same always-on family as host-your-agent); pulled toward L6 because it's a standing,
  self-updating, spec-shaped client artifact rather than a one-off report. Flagged for your
  final call — if you'd rather it read as a clean L5 to sit inside the "Always-on (L5)" group,
  that's a one-word change.
- **Placeholder data is obviously fictional** (Northwind Digital / Riverside Dental Group) and
  contains no PII and no real client. The template renders a full demo on its own so a member
  sees the shape before editing — but every value is flagged "replace entirely."
- **Verified runnable:** both JSON payloads parse; the template references zero external
  URLs (fully self-contained); brand tokens, both data fences, the render script, and the
  horizontal-scroll wrapper on the rankings table are all present.
