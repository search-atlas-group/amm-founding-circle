# Repository Instructions

This repo is for Agentic Marketing Mastermind members. Keep it teaching-first,
and clearly label any advanced source material that needs local adaptation.

## Content Boundary

- Do not add employee-only deployment systems, unpublished customer data,
  credentials, or token locations.
- Prefer examples that work against public tools, local files, GitHub-style pull
  requests, pasted transcripts, or clearly documented optional services.
- Keep commands safe to run on a member laptop. If a command changes files,
  make the affected path explicit.
- Never commit real tokens, cookies, browser profiles, session dumps, or member
  data.

## Tone

- Prefer concrete playbooks over theory.
- Explain why a rule exists before giving commands.
- Keep advanced orchestration concepts as optional patterns, not prerequisites.
